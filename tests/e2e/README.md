# E2E harness

Playwright + `office-addin-test-server` / `office-addin-test-helpers` driving
real Word. These tests do NOT run in CI or in `pnpm test` — they require:

1. **A local macOS with Microsoft Word installed** (M365 or 2019+).
2. **Playwright browsers installed locally**:
   ```bash
   pnpm playwright install --with-deps chromium
   ```
3. **The add-in sideloaded into Word** via `office-addin-debugging` —
   `office-addin-test-server` automates test-result collection from the
   running add-in, but first-run may need a one-time permission click.

## Run the E2E harness

From the fair-copy root:

```bash
pnpm playwright test
```

## What's stubbed (DEFERRED from M2 T12 scaffolding)

The following helpers in `helpers.ts` currently throw `not yet wired`:

- `sideload(manifestPath)` — needs `office-addin-debugging` + test server wiring.
- `openDocument(docxPath)` — needs `office-addin-test-helpers` wiring or
  AppleScript fallback on macOS.
- `clickCleanButton()` — needs CDP attachment to the task pane iframe.
- `getDocumentState()` — needs an Office.js readback script + postMessage bridge.

These are filled in during **manual verification** (M2 T13) when the Word
install is available. The test skeletons in `smoke.e2e.spec.ts` show the
expected call shape so wiring the helpers should be a straight path.

## Fixture regeneration

Fixtures are generated by `tests/e2e/fixtures/seed.ts`:

```bash
npx tsx tests/e2e/fixtures/seed.ts
```

Note: the `tracked-changes-doc.docx` and `image-heavy-doc.docx` fixtures are
currently placeholders — the `docx` npm library has limited tracked-changes
and image-injection support. During manual verification, replace those
fixtures with authored-in-Word samples to get realistic test coverage.
